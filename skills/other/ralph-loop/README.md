# Ralph Loop

Autonomous implementation loop. Each iteration spawns a fresh Claude process that picks one task from a PRD, implements it with TDD, commits, and exits. Progress lives in git, not context.

Based on [Geoffrey Huntley's Ralph Wiggum loop](https://ghuntley.com/ralph/) and [Matt Pocock's adaptation](https://github.com/mattpocock/ralph-workshop-repo-001).

## Prerequisites

- `plans/<name>.md` — your PRD (create with `/prd-create`)
- `plans/prompt-<name>.md` — generated by running `/ralph-loop` in Claude

## Setup

Run `/ralph-loop` inside Claude Code. It will:

1. List available PRDs and ask which one to use (or accept a name as argument)
2. Detect your project's feedback loops (test, typecheck, lint)
3. Generate `plans/prompt-<name>.md` with the right commands
4. Print the commands to start the loop

## Running

### Single iteration (human in the loop)

```bash
bash ~/.almanac/skills/ralph-loop/scripts/once.sh auth-system
```

Runs one pass. You review the result before continuing. Good for:
- First iteration (sanity check the approach)
- After an abort (diagnose and unblock)
- When you want to steer

### Autonomous (AFK)

```bash
bash ~/.almanac/skills/ralph-loop/scripts/afk.sh auth-system 10
```

Runs 10 iterations autonomously. Stops early when:
- All tasks complete (`COMPLETE` signal)
- A task is blocked (`ABORT` signal)

### Monitoring progress

```bash
git log --grep="RALPH(auth-system)" --oneline
```

Each `RALPH(auth-system):` commit message contains what was done and notes for the next iteration. The name prefix means multiple PRDs can run against the same repo without confusing each other's progress.

## How it works

```
afk.sh auth-system 10
  └── for each iteration:
        1. Read last 10 RALPH(auth-system): commits from git log
        2. Spawn fresh `claude --print @plans/prompt-auth-system.md`
        3. Claude reads PRD, breaks into tasks, picks next unfinished one
        4. Implements with TDD, runs feedback loops
        5. Commits with RALPH(auth-system): prefix
        6. Exits → next iteration
```

No subagents, no shared memory. Each iteration is a separate OS process with fresh context. Git is the only state that carries between iterations.

## Files

```
plans/
├── brief.md                ← from /grill-me (optional)
├── auth-system.md          ← from /prd-create (the PRD)
├── prompt-auth-system.md   ← from /ralph-loop setup
├── dashboard.md            ← another PRD (can coexist)
└── prompt-dashboard.md     ← its own prompt
```

## Resuming

Just run `afk.sh` again with the same name. It reads `RALPH(<name>):` commits to know what's already done.
