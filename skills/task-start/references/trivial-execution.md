# Trivial Execution

The task is small and well-defined. Get it done directly.

## Steps

1. **Explore** — Read the relevant files. Understand the current behavior and where the change goes.
2. **Implement** — Make the change. Keep it minimal and focused.
3. **Verify** — Run existing tests (`test`, `check`, `lint` — whatever the project uses). If no test runner exists, manually verify the change works.
4. **Report** — State what was changed and why, in 2-3 sentences.

## Rules

- No formal plan needed
- No step list needed — just do the work
- If you discover mid-task that this is actually more complex than expected, stop and re-assess using the complexity heuristic. Upgrade to moderate or complex execution as needed.
