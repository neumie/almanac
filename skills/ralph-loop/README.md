# Ralph Loop

Autonomous implementation loop. Each iteration spawns a fresh Claude process that picks one task from the PRD, implements it with TDD, commits, and exits. Progress lives in git, not context.

Based on [Geoffrey Huntley's Ralph Wiggum loop](https://ghuntley.com/ralph/) and [Matt Pocock's adaptation](https://github.com/mattpocock/ralph-workshop-repo-001).

## Prerequisites

- `plans/prd.md` — your PRD (create with `/prd-create`)
- `plans/prompt.md` — generated by running `/ralph-loop` in Claude

## Setup

Run `/ralph-loop` inside Claude Code. It will:

1. Detect your project's feedback loops (test, typecheck, lint)
2. Generate `plans/prompt.md` with the right commands
3. Print the commands to start the loop

## Running

### Single iteration (human in the loop)

```bash
bash ~/.almanac/skills/ralph-loop/scripts/once.sh
```

Runs one pass. You review the result before continuing. Good for:
- First iteration (sanity check the approach)
- After an abort (diagnose and unblock)
- When you want to steer

### Autonomous (AFK)

```bash
bash ~/.almanac/skills/ralph-loop/scripts/afk.sh 10
```

Runs 10 iterations autonomously. Stops early when:
- All tasks complete (`COMPLETE` signal)
- A task is blocked (`ABORT` signal)

### Monitoring progress

```bash
git log --grep="RALPH" --oneline
```

Each `RALPH:` commit message contains what was done and notes for the next iteration.

## How it works

```
afk.sh
  └── for each iteration:
        1. Read last 10 RALPH: commits from git log
        2. Spawn fresh `claude --print @plans/prompt.md`
        3. Claude reads PRD, breaks into tasks, picks next unfinished one
        4. Implements with TDD, runs feedback loops
        5. Commits with RALPH: prefix
        6. Exits → next iteration
```

No subagents, no shared memory. Each iteration is a separate OS process with fresh context. Git is the only state that carries between iterations.

## Files

```
plans/
├── brief.md      ← from /grill-me (optional)
├── prd.md        ← from /prd-create (required)
└── prompt.md     ← from /ralph-loop setup (required)
```

## Resuming

Just run `afk.sh` again. It reads `RALPH:` commits to know what's already done.
